TeamCity Build Server: Understanding Unit Testing and Code Coverage Analysis in TeamCity

Project Overview: What is CI?

CI allows for multiple integrations of code into a shared repository. Each check in can be used to verify problems early on in the development process. The build can be automated and test itself, and anyone who has access to the server can see what's going on

Benefits of CI

-Developers check out code into their local workspaces, then commit changes to a repository after code review. -The CI server monitors the repository and checks out changes, then builds the system and runs unit/integration tests -The team is informed of the build's success or failure status, and the team can fix the issue quickly and efficiently
